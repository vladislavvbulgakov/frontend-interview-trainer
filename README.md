# FrontendLoop

**Проект находится в активной разработке**

FrontendLoop — веб-приложение для симуляции реального процесса найма фронтенд-разработчиков с персональной аналитикой и метриками уверенности.

## Текущий стек технологий
- JavaScript
- React
- HTML
- CSS Modules
- Vite
- Axios
- Node.js
- Express
- PostgreSQL
- REST API
---

## Цель продукта

Помочь фронтенд-разработчикам:
- оценивать свой текущий уровень
- понимать сильные и слабые стороны
- целенаправленно улучшать уверенность по выбранному грейду
- готовиться к собеседованиям системно, а не хаотично

## Идея приложения

FrontendLoop позволяет пользователю проходить симулированные собеседования, максимально приближённые к реальным процессам найма:

- HR-интервью
- Технические интервью
- Полный цикл (HR + Technical)

С возможностью выбора:
- грейда (Junior / Middle / Senior)
- тем технических вопросов (HTML/CSS, JavaScript, React, HTTP/REST и др.)

Каждое интервью всегда завершается финальным решением:
- ❌ **No Hire**
- ✅ **Hire**

Даже при решении *No Hire* пользователь может видеть положительную динамику своей подготовки и роста уверенности.

## Loop Score

**Loop Score** — это агрегированная метрика, отражающая динамику уверенности пользователя в процессе прохождений интервью.

### Как планируется работа Loop Score:
- Считается для конкретного грейда
- LoopScore = **min(Целостность прохождений интервью, Стабильность знаний (в будущем  `UserQuestionStat.confidence_score`, Коэффицент грейда)**  
- отслеживает прогресс по темам
- реагирует на положительную и отрицательную динамику

Данная метрика позволит наблюдать общую траекторию развития.

## Инструкция по запуску

**Убедитесь, что установлены:**
- Node.js 18+
- npm (устанавливается вместе с Node.js)
- PostgreSQL (если хотите поднять БД локально, на текущем этапе можете воспользоваться mock-данными)
1. Клонируйте репозиторий в удобную папку на устройстве
    ```code
    git clone https://github.com/vladislavvbulgakov/frontend-interview-trainer.git
    cd frontend-interview-trainer
    ```
2. Запуск клиента
    ```code
    cd client
    npm install
    npm run dev
    ```
  3. Работа с mock-данными
      - В текущем состоянии проекта, если Вы не планируете поднимать БД локально, то для демонстрации тем вопросов и их карточек необходимо воспользоваться mock-данными по пути:
     ```code
     client/src/app/mockData/
                            mockTopics.jsx
                            mockQuestions.jsx
     ```
      - Далее перейдите в компоненты KnowledgeBase.jsx и StartInterview.jsx и замените кастомные хуки на mock-данные.
  4. Запуск сервера (Backend)
      - При запуске сервера база данных автоматически инициализируется и наполняется начальными данными (topics и questions).
      - На текущем этапе разработки backend предполагает использование локальной PostgreSQL базы данных.
    
        4.1 Подготовка базы данных
          1. Убедитесь, что PostgreSQL установлен и запущен.
          2. Создайте пустую базу данных, например:
          ```code
          CREATE DATABASE frontendloop;
          ***
          Название базы данных может быть любым — оно должно совпадать со значением DB_NAME в файле .env.
          ```

        4.2 Настройка переменных окружения
          - В папке server находится файл .env.example.
            1. Скопируйте его и переименуйте в .env:
            ```code
            cd server
            cp .env.example .env
            ```
            2. Отредактируйте файл .env под своё окружение:
            ```code
            PORT=3030
            
            DB_HOST=localhost
            DB_PORT=5432
            DB_USER=postgres
            DB_PASSWORD=password
            DB_NAME=frontendLoop
            
            # Автоматическая инициализация БД при старте сервера
            DB_AUTO_SEED = true
            DB_RESET = true
            
            FRONTEND_URL=http://localhost:5173
            FRONTEND_URL_PRODUCTION=http://frontendloop.ru/
            
            # Mock user для локальной разработки (используется до внедрения auth)
            MOCK_USER_ID=2ff6430e-03b8-42b2-81dd-7eb7a184092e
            ```

        4.3 Запуск 
        - Из папки server выполните:
        ```code
        npm install
        npm run dev
        ```
        - После успешного запуска вы увидите сообщение вида:

      5 Проверка работы
        - После запуска:
        ```code
        Frontend:
        http://localhost:5173
        
        Backend:
        http://localhost:3030/
        ```

## Планируемые возможности (MVP)

- выбор типа интервью (HR / Technical / Full)
- выбор грейда
- выбор тем технических вопросов
- генерация случайных вопросов из базы
- ответы на вопросы:
  - «Уверенно»
  - «Сомневаюсь»
  - «Не знаю»
- подсказки в HR-интервью
- просмотр эталонных ответов (во время технической секции)
- аналитика прогресса:
  - процент успешности
  - тренд (↑ / → / ↓)
  - последний результат
- отчёт после интервью:
  - общий score
  - финальное решение (Hire / No Hire)
  - рекомендации по темам
- история интервью
- база популярных вопросов с персональными статусами
- авторизация и профиль пользователя
